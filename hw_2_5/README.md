# Домашнее задание 2.5: Синтетические данные через Stable Diffusion + ControlNet

Аугментация датасета через генеративные модели: добавляем синтетические изображения «редких» классов и измеряем эффект на классификаторе.

## TL;DR

- **Задача:** классификация crops из COCO subset на 10 классов (тех же, что в части 1).
- **Редкие классы:** `cat` (101 crop) и `bicycle` (81 crop) — самые низкие AP по результатам части 1.
- **Метод:** SD 1.5 + ControlNet (Canny). Canny берётся с реальных crops → задаёт композицию; промпт варьируем для разнообразия.
- **Сгенерировано:** 202 cat + 162 bicycle = **364 синтетических изображения**.
- **Ablation:** baseline (только реальные) | synth_1× | synth_2×.

**Главный результат:** overall accuracy практически не меняется (~95%), но per-class accuracy для `cat` поднимается с **96.3% → 100%** в synth_2×. Эффект на bicycle нестабилен из-за низкой статистики (всего 17 val примеров).

## Метод генерации

**Pipeline для каждого синтетического примера:**
```
real_crop → resize 512×512 → Canny(100, 200) ──→ ControlNet
                                                     │
random_prompt + neg_prompt → SD 1.5 (UniPC, 25 steps, guidance=7.5)
                                                     │
                                                     ▼
                                              synthetic_image (512×512)
```

## Наблюдения

### 1. Синтетика помогает целевому классу — но только в достаточном объёме
- **`cat`:** synth_1× не меняет accuracy (0.963 → 0.963), а synth_2× выводит её на **1.000** (все 27 val примеров правильно классифицированы). То есть **доза имеет значение**: единичное добавление синтетики недостаточно, нужно ≥2× от исходного объёма реальных данных.
- **`bicycle`:** в synth_1× accuracy неожиданно **падает на 11.8pp** (0.882 → 0.765), а в synth_2× возвращается на исходный уровень. Это объясняется маленьким val (17 примеров — одна ошибка = -5.9pp) — высокая дисперсия делает оценку нестабильной.

### 2. Эффект «переливается» на соседние классы
- **`motorcycle` +7.4pp** в synth_1× и synth_2× — модель лучше отделяет велосипеды от мотоциклов после того, как увидела больше визуальных вариаций двухколёсного транспорта.
- **`car` +8.1pp** и **`bird` +9.1pp** в synth_2× — менее очевидно, возможно эффект регуляризации (больше данных в целом ≈ меньше переобучения).

### 3. Есть и побочные эффекты
- **`dog` падает с 0.886 до 0.750 в synth_2×** — это **classification trade-off**: модель учится так чётко отделять «cat», что начинает путать собак (например, маленьких пушистых) с другими классами.
- Это типичный паттерн при балансировке датасета — улучшение одного класса часто приходит за счёт похожих.

### 4. Val loss в synth_2× ниже всех — модель лучше калибрована
Кривые val loss (`ablation_curves.png`):
- synth_2× выходит на **0.144** (лучшее значение)
- baseline застревает на **0.176**
- synth_1× — самый шумный, 0.192

То есть даже там, где accuracy одинаковая, synth_2× **более уверен в правильных предсказаниях**.

## Выводы

1. **Стоит ли использовать SD+ControlNet синтетику?** Да, но с оговорками: эффект заметен только при достаточном объёме (≥2× от реальных) и для классов с действительно низким числом примеров.

2. **Какой класс выиграл больше всего?** `cat`: с 96.3% до 100% per-class accuracy при добавлении 2× синтетики. У `bicycle` результат «нулевой эффект» из-за малого val — нужна более крупная валидация для надёжной оценки.

3. **Есть ли риски?** Да — модель может «переучиться» на синтетику и ухудшить классификацию визуально похожих классов (наш `dog` -13.6pp при synth_2× — это плата за идеальный cat).
